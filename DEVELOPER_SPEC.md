# 1. 项目基本信息

## 1.1 项目名称

**市长模拟器：城市账本**

```text
Mayor Simulator: City Ledger
```

```text
mayor-simulator
```

## 1.2 产品类型

- 单机网页游戏；
- 城市政策模拟；
- 回合制文字策略；
- 数据和事件驱动；
- 无后端；

- 无联网要求；

## 1.3 产品定位

玩家担任中国虚构城市“安澜市”的市级主要负责人，在五年任期内，通过预算安排、政策选择、项目建设和事件处理，平衡以下目标：

- 居民生活质量；
- 经济与就业；
- 财政可持续性；
- 公共服务公平性；
- 生态环境与城市安全；

- 片区发展差异；
- 外部环境和突发事件。

# 5. 回合与流程

## 5.1 时间规则

- 一局共20个季度；
- 一个季度为一个回合；
- 每4个季度为一年；
- 共5年；
- 当前季度用“第X年·第X季度”显示。

季度序号对应：

```text
第1季度 = 第1年第1季度
第4季度 = 第1年第4季度
第5季度 = 第2年第1季度
第20季度 = 第5年第4季度
```

## 5.2 游戏阶段

使用明确的状态机，至少包含：

```ts
type GamePhase =
  | 'YEAR_PLANNING'
  | 'BRIEFING'
  | 'EVENT_DECISION'
  | 'ACTION'
  | 'SIMULATING'
  | 'QUARTER_REPORT'
  | 'YEAR_REPORT'
  | 'GAME_END';
```

不得仅依靠页面判断流程。

## 5.3 每季度流程

### 第一步：年度规划

只在每年第1季度出现。

玩家必须：

- 确认本年度季度预算；
- 选择2项年度重点目标；
- 决定是否提交融资申请。

### 第二步：季度简报

展示：

- 核心指标；
- 上季度变化；
- 风险预警；
- 政策进度；
- 财政压力；
- 本季度待处理事件。

### 第三步：处理事件

- 本季度有待处理事件时，必须先处理；
- 每季度最多出现2个普通事件；
- 强制危机事件不受普通上限限制；
- 所有事件处理完毕后，才能进入政策操作阶段。

### 第四步：政策与项目操作

玩家可以：

- 启动政策；
- 查看政策；
- 暂停政策；
- 恢复政策；
- 取消尚未完成的政策；
- 查看未来行政能力占用；
- 查看资金承诺。

### 第五步：结束季度

结束前弹出确认框，显示：

- 预计本季度支出；
- 预计行政能力使用；
- 高风险项目；
- 可能出现的资金缺口；
- 尚未处理的重要提醒。

### 第六步：季度结算

模拟引擎运行，并生成季度报告。

### 第七步：季度报告

显示：

- 指标变化；
- 变化原因；
- 财政报表；
- 政策进展；
- 事件后果；
- 四季度预测。

每年第四季度结束后，再显示年度报告。

# 6. 年度目标系统

每年玩家必须从以下目标中选择2项：

| ID | 目标 | 年度判断标准 |
|---|---|---|
| `employment` | 稳定就业 | 年末失业率不高于年初，或下降至少0.3个百分点 |
| `education` | 改善教育 | 教育综合分提高至少3分，且最差片区不继续恶化 |
| `healthcare` | 改善医疗 | 医疗综合分提高至少3分 |
| `mobility` | 缩短通勤 | 全市平均通勤时间下降至少1分钟 |
| `fiscal` | 控制财政风险 | 无新增欠款，偿债压力不恶化超过2个百分点 |
| `environment` | 提升生态安全 | 环境安全综合分提高至少3分 |

年度目标不提供隐藏数值奖励，只代表公开承诺。

年末结算：

- 每完成1项目标：政府公信力 `+2`；
- 每失败1项目标：政府公信力 `-2.5`；
- 因外部重大灾害失败，也不完全免除影响，但失败惩罚降低为 `-1.5`；
- 结果写入年度报告和因果账本。

# 7. 行政能力系统

## 7.1 基础规则

标准难度下：

```text
每季度基础行政能力 = 12点
```

实际行政能力：

```text
实际行政能力 = floor(基础行政能力 × 行政效率指数 ÷ 100)
```

限制：

```text
最低8点，最高16点
```

## 7.2 行政能力用途

以下行为消耗行政能力：

- 政策审批；
- 项目实施；
- 重大事件处理；
- 专项检查；
- 复杂协调；
- 临时应急任务。

## 7.3 超载规则

允许玩家在警告后，将承诺行政能力提高到：

```text
本季度行政能力 + 4
```

如果承诺量超过实际行政能力：

```text
执行效率 = 实际行政能力 ÷ 已承诺行政能力
```

政策和项目进度按执行效率增加，不使用纯随机延误。

例如：

```text
实际能力 = 12
承诺能力 = 15
执行效率 = 0.8
```

原本需要1季度的进度，本季度只推进0.8。

行政超载同时产生：

- 政府日常服务效率下降；
- 项目延期风险提醒；
- 连续超载时公信力下降；
- 合规风险上升。

# 8. 政策生命周期

## 8.1 政策状态

```ts
type PolicyStatus =
  | 'APPROVING'
  | 'IMPLEMENTING'
  | 'ACTIVE'
  | 'PAUSED'
  | 'COMPLETED'
  | 'CANCELLED'
  | 'EXPIRED';
```

## 8.2 政策阶段

```text
启动
→ 审批
→ 实施或建设
→ 生效运营
→ 完成或到期
```

## 8.3 政策成本字段

每项政策必须包含：

- 启动成本；
- 总资本成本；
- 审批季度数；
- 实施季度数；
- 每季度行政能力消耗；
- 年度运营成本；
- 生效持续时间；
- 前置条件；
- 预测效果区间；
- 数据可信度；
- 风险；
- 受益群体；
- 可能受损群体；
- 是否可取消；
- 取消代价；
- 是否可重复。

资本成本默认在实施阶段平均支付。

例如：

```text
总资本成本12亿元
实施周期4季度
每季度资本支出3亿元
```

年度运营成本转换为：

```text
季度运营成本 = 年度运营成本 ÷ 4
```

## 8.4 资金不足

如果无法支付资本支出：

- 项目本季度暂停；
- 不增加建设进度；
- 产生“资金不到位”记录；
- 连续2季度资金不到位，公信力下降；
- 连续3季度资金不到位，可能触发项目处置事件。

如果无法支付运营支出：

- 政策效果按实际支付比例折算；
- 公共服务类设施不得在无运营经费时保持完整效果。

## 8.5 取消政策

审批或建设中的项目可以取消。

取消后：

- 已支付资金不退还；
- 未支付资本成本取消；
- 行政能力从下一季度释放；
- 根据完成进度扣除公信力；
- 可能触发后续事件；
- 已建成的部分资产可能形成维护成本。

## 8.6 政策效果原则

政策不得直接修改以下最终指标：

- GDP；
- 失业率；
- 全市满意度；
- 最终评分。

政策必须优先修改中间变量，例如：

- 产业生产率；
- 企业投资意愿；
- 岗位容量；
- 学位数量；
- 教师充足度；
- 医疗服务能力；
- 公交可达性；
- 排水能力；
- 住房供应；
- 污染强度；
- 行政效率；
- 维护效率。

最终指标由模拟引擎统一计算。

# 11. 随机性与可复现性

## 11.1 固定随机数生成器

实现独立的种子随机数生成器，例如：

```text

```

必须支持：

- `next()`；
- 指定范围随机；
- 正态分布；
- 加权选择；

禁止核心模拟直接使用：

```ts
Math.random()
```

## 11.2 存档要求

存档必须保存：

- 初始种子；
- 当前随机数状态；
- 当前季度；
- 所有政策状态；
- 所有事件状态；
- 所有延迟效果；
- 所有财政债务记录。

## 11.3 预测随机数隔离

玩家查看预测时，不得消耗正式游戏的随机数状态。

预测使用：

```text
hash(游戏种子, 当前季度, "forecast")
```

生成独立随机流。

打开或关闭预测页面不得改变未来结果。

## 15.2 初始季度收入

```text

转移和专项资金：11亿元
```

征管效率由行政效率影响，但修正范围应控制在：

## 15.5 初始债务

```text

初始季度偿还本金：7亿元
```

当季利息：

```text
期初债务余额 × 0.8%
```

融资增加现金和债务余额，不计入财政收入。

## 15.6 新增融资

玩家只能在每年第1季度提交融资申请。

可选规模：

```text
0、10、20、30、40亿元
```

审批上限：

| 偿债支出占可用收入比例 | 本年度最高可申请 |
|---|---:|
| 低于15% | 40亿元 |
| 15%—20% | 20亿元 |
| 高于20% | 不允许普通新增融资 |

融资：

- 延迟1季度到账；
- 期限20季度；
- 按季度等额偿还本金；
- 季度利率0.8%；
- 记录独立债务批次；
- 超过本局时间的偿还义务仍需计入期末风险评估。

## 15.7 支付优先级

每季度资金支付顺序：

1. 债务本息；
2. 旧欠款偿还；
3. 基础公共服务预算；
4. 已生效政策运营支出；
5. 历史项目资本支出；
6. 新政策和新项目资本支出；
7. 非必要临时支出。

如果资金不足：

- 不允许现金小于0；
- 未支付金额进入欠款；
- 未支付项目资本支出导致项目暂停；
- 公共服务按实际支付比例计算效果；
- 欠款必须进入风险和公信力计算。

## 15.8 财政报表守恒

必须满足：

```text
期末现金
= 期初现金
+ 实际财政收入
+ 融资到账
- 实际支付支出
```

同时满足：

```text
期末债务
= 期初债务
+ 新增融资
- 已偿还本金
```

因四舍五入造成的显示差异不得影响内部计算。

# 19. 产业与就业计算

## 19.1 产业需求

建议基准公式：

```text
产业需求增速
= 产业敏感度 × 宏观需求变化
+ 政策修正
+ 产业随机扰动
```

产业随机扰动应小于宏观冲击，且由种子随机数生成。

## 19.2 产业产出

建议公式：

```text
目标产出
= 基准产出
× (需求指数 ÷ 100) ^ 0.55
× (生产率指数 ÷ 100) ^ 0.70
× (投资指数 ÷ 50) ^ 0.15
× 基础设施修正
× 劳动力约束
```

实际产出平滑：

```text
本期产出
= 上期产出 × 0.65
+ 目标产出 × 0.35
```

## 19.3 就业需求

```text
目标就业
= 基准就业
× (本期产出 ÷ 基准产出) ^ 就业弹性
× (100 ÷ 生产率指数) ^ 0.25
× 岗位容量修正
```

实际就业：

```text
本期就业
= 上期就业 × 0.70
+ 目标就业 × 0.30
```

## 19.4 劳动力人口

```text
劳动人口
= 总人口
× 劳动年龄人口比例
× 劳动参与率
```

## 19.5 失业率

```text
失业率
= max(0, 劳动人口 - 实际就业)
÷ 劳动人口
```

将失业率限制在合理显示区间，但不得掩盖真实经济恶化。

## 19.6 工资指数

工资变化受以下因素影响：

- 生产率增长；
- 劳动力紧张程度；
- 产业结构；
- 通胀或外部环境；
- 技能匹配。

建议季度变化限制：

```text
-1.5% 至 +2.5%
```

# 20. 人口系统

## 20.1 基础公式

```text
下期人口
= 当前人口
+ 出生
- 死亡
+ 净迁入
```

季度基准：

```text
出生率约0.18%
死亡率约0.20%
```

可根据老龄化略微调整。

## 20.2 城市吸引力

城市吸引力由以下维度组成：

| 维度 | 权重 |
|---|---|
| 就业机会 | 32% |
| 实际工资 | 18% |
| 住房负担 | 18% |
| 公共服务 | 17% |
| 通勤 | 8% |
| 环境 | 7% |

建议净迁移公式：

```text
净迁入
= 总人口
× clamp(
    (吸引力分数 - 58) × 0.00009,
    -0.004,
    0.004
  )
+ 小幅随机扰动
```

## 20.3 片区迁移

迁入人口根据片区吸引力分配。

片区吸引力至少考虑：

- 房租；
- 空置住房；
- 就业可达性；
- 教育；
- 医疗；
- 交通；
- 环境。

迁出人口优先来自吸引力较低的片区。

不得出现无原因的单季度片区人口暴涨。

# 21. 住房系统

## 21.1 住房需求

```text
家庭住房需求
= 总人口 ÷ 平均家庭规模
```

初始平均家庭规模：

```text
2.35人
```

## 21.2 租金

片区租金变化建议：

```text
住房压力
= 住房需求 ÷ 可入住住房

租金增速
= clamp(
    0.035 × (住房压力 - 1)
    + 0.20 × 工资增速
    + 片区吸引力修正,
    -0.03,
    0.03
  )
```

## 21.3 房价

房价变化受以下因素影响：

- 租金；
- 空置率；
- 居民收入；
- 商业信心；
- 新增住房供应；
- 土地和住房政策；
- 外部环境。

房价不得只涨不跌。

## 21.4 空置率

空置率范围：

```text
2%—25%
```

空置率应根据供需压力逐季度平滑变化。

## 21.5 保障性住房

保障性租赁住房和青年住房应：

- 增加特定群体可负担住房供给；
- 降低青年租住者住房压力；
- 产生建设成本；
- 产生持续运营成本；
- 不得直接让全市房价大幅下降。

# 22. 公共服务系统

## 22.1 通用规则

公共服务有效水平：

```text
有效服务水平
= 设施容量
× 人员充足度
× 运营经费充足度
```

不得只建设建筑就自动获得完整效果。

## 22.2 教育

教育至少考虑：

- 学位；
- 学生人数；
- 教师充足度；
- 教育预算；
- 片区差异。

建议教育得分：

```text
教育得分
= clamp(
    70
    + 35 × (学位学生比 - 1)
    + 20 × (教师指数 ÷ 100 - 1)
    + 20 × (预算充足率 - 1),
    20,
    95
  )
```

## 22.3 医疗

医疗至少考虑：

- 人口；
- 老龄人口；
- 医疗设施能力；
- 医护人员；
- 医疗预算；
- 季节性疾病；
- 重大事件。

输出：

- 医疗压力；
- 医疗综合分；
- 片区医疗差距。

## 22.4 养老与托育

养老和托育分别影响：

- 老年居民；
- 有孩家庭；
- 青年家庭留城意愿；
- 社会服务预算；
- 相关运营成本。

# 23. 交通系统

不制作道路地图，采用片区可达性模型。

## 23.1 主要指标

- 交通可达性；
- 道路条件；
- 拥堵指数；
- 平均通勤时间；
- 公交服务能力；
- 货运效率；
- 公交运营补贴。

## 23.2 通勤时间

建议公式：

```text
平均通勤时间
= clamp(
    18
    + 0.28 × 拥堵指数
    - 0.08 × 交通可达性,
    15,
    65
  )
```

全市通勤时间按照片区人口加权。

## 23.3 交通政策原则

- 扩建道路可以短期改善道路条件；
- 道路改善可能诱发新增出行需求；
- 公交优先可能降低部分路段机动车通行能力；
- 公交改善主要利好青年、产业工人和低收入居民；
- 公交线路需要持续运营补贴；
- 停车管理可能改善拥堵，但引发部分居民和商户不满。

# 24. 环境、安全与灾害

## 24.1 环境

空气和环境质量受以下因素影响：

- 工业产出；
- 产业排放强度；
- 物流运输；
- 污染治理政策；
- 气象扰动；
- 环境预算。

环境指标高值代表更好。

## 24.2 洪涝风险

建议公式：

```text
洪涝风险
= 洪涝暴露
× (1 - 排水能力 ÷ 120)
× 季节天气系数
× 设施脆弱度修正
```

风险归一化为0—100，高值代表危险。

第二、第三季度应提高强降雨事件概率。

## 24.3 安全生产

安全风险受以下因素影响：

- 工业和物流活动强度；
- 安全能力；
- 设施老化；
- 行政超载；
- 专项检查；
- 项目施工数量。

安全检查可以短期降低风险，但：

- 消耗行政能力；
- 可能影响企业正常生产；
- 不产生永久性完全免疫。

## 24.4 基础设施折旧

道路、管网、排水和公共设施需要维护。

维护预算不足时：

```text
季度折旧
= 0.35 × (1 - 维护预算充足率)
```

维护预算高于需求时可以小幅恢复，但单季度恢复不超过0.1—0.2个指数点，专项政策除外。

# 25. 居民感受和政府公信力

## 25.1 群体感受

每个群体单独计算，不设置唯一的“全体居民满意度”作为核心依据。

群体目标感受：

```text
目标感受
= 各评价维度得分 × 对应权重之和
+ 临时预期影响
+ 公平感修正
```

平滑更新：

```text
本期感受
= 上期感受 × 0.65
+ 目标感受 × 0.35
```

限制在：

```text
0—100
```

## 25.2 政府公信力

政府公信力受以下因素影响：

- 各群体感受；
- 政策承诺兑现；
- 项目按期完成；
- 项目延期；
- 财政欠款；
- 行政超载；
- 事件处理；
- 年度目标完成情况；
- 片区公平性。

公信力不得等同于居民感受。

```ts
interface Contribution {
```

# 32. 数据结构

## 32.1 游戏状态

至少包含：

```ts
interface GameState {
  schemaVersion: number;
  gameId: string;
  createdAt: string;
  updatedAt: string;

  seed: number;
  rngState: number;
  difficulty: 'EASY' | 'STANDARD' | 'HARD';

  turn: {
    quarterIndex: number;
    year: number;
    quarterOfYear: number;

  };

  city: CityState;

  policies: PolicyInstance[];

  pendingEvents: PendingEvent[];

  reports: QuarterReport[];

```

## 32.2 政策定义

```ts
interface PolicyDefinition {
  id: string;
  name: string;
  category: string;
  description: string;
  authorityDescription: string;

  targetDistricts: string[];
  beneficiaries: string[];
  negativelyAffectedGroups: string[];

  startupCost: number;
  capitalCost: number;
  annualOperatingCost: number;

  approvalQuarters: number;
  implementationQuarters: number;
  durationQuarters: number | null;
  adminCostPerQuarter: number;

  repeatable: boolean;
  cancellable: boolean;
  cooldownQuarters: number;

  prerequisites: ConditionNode;
  effects: EffectDefinition[];

  confidence: 'HIGH' | 'MEDIUM' | 'LOW';
  risks: string[];
  causalChain: string[];
}
```

## 32.3 条件系统

不得使用代码字符串和 `eval`。

使用结构化条件：

```ts
type ConditionNode =
  | {
      type: 'ALL';
      conditions: ConditionNode[];
    }
  | {
      type: 'ANY';
      conditions: ConditionNode[];
    }
  | {
      type: 'NOT';
      condition: ConditionNode;
    }
  | {
      type: 'COMPARE';
      path: string;
      operator: 'GT' | 'GTE' | 'LT' | 'LTE' | 'EQ' | 'NEQ';
      value: number | string | boolean;
    };
```

`path`必须来自白名单。

## 32.4 效果系统

```ts
interface EffectDefinition {
  id: string;
  stage: 'ON_START' | 'ON_COMPLETE' | 'WHILE_ACTIVE' | 'ON_EXPIRE';
  scope: 'CITY' | 'DISTRICT' | 'INDUSTRY' | 'GROUP';
  scopeId?: string;
  metric: string;
  mode: 'ADD' | 'MULTIPLY';
  minValue: number;
  maxValue: number;
  delayQuarters: number;
  durationQuarters: number | null;
  decayRate?: number;
  description: string;
}
```

政策启动时，根据种子为效果生成实际值。

可信度建议：

| 可信度 | 预测区间 |
|---|---|
| 高 | 中心值上下约10% |
| 中 | 中心值上下约20% |
| 低 | 中心值上下约30% |

实际结果在启动时确定并保存，避免读档后变化。

# 38. 测试要求

## 38.1 单元测试

至少覆盖：

1. 相同种子产生相同结果；
2. 查看预测不改变正式随机数状态；
3. 财政现金守恒；
4. 债务余额守恒；
5. 融资不计入财政收入；
6. 政策审批和实施生命周期；
7. 资金不足导致项目暂停；
8. 行政超载降低进度；
9. 运营经费不足降低政策效果；
10. 事件条件判断；
11. 事件冷却；
12. 后续事件调度；
13. 年度目标结算；
14. 居民权重合计；
15. 最终评分；
16. 存档序列化和恢复；
17. 导入非法存档失败；
18. 第20季度正常结束；
19. 因果贡献与指标变化一致；
20. 所有指标不存在NaN和Infinity。

## 38.2 数据测试

必须自动检查：

- 政策和事件数量；
- 引用完整；
- 数值范围；
- 权重；
- 效果字段；
- 保底事件选项；
- 无重复ID。

## 38.3 批量模拟

编写：

```bash
npm run sim:batch
```

至少提供三种自动策略：

### 无操作策略

- 不启动新政策；
- 使用默认预算；
- 事件选择成本最低方案。

### 稳健策略

- 优先财政安全；
- 优先维护和公共服务；
- 控制大型项目；
- 偿债压力过高时不融资。

### 扩张策略

- 优先经济项目；
- 更愿意融资；
- 同时推进更多项目；
- 接受较高财政风险。

正式验收前至少运行：

```text
1000个种子 × 20季度
```

最低要求：

- 无崩溃；
- 无NaN；
- 无Infinity；
- 无负人口；
- 无负债务；
- 无负住房；
- 现金不低于0；
- 所有指数处于定义范围；
- 所有局均能到达结局；
- 不得100%出现同一种结局；
- 标准难度稳健策略不应普遍陷入必然财政崩溃；
- 扩张策略应明显比稳健策略具有更高债务风险。

结果写入：

```text
BALANCE_REPORT.md
```
