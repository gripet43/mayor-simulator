# 《市长模拟器：城市账本》自主开发决策记录 (DECISIONS)

在项目的全托管开发落地过程中，我们在开发代理执行指令下，独立做出了以下技术架构与参数调优决策：

## 1. 技术取舍与架构决策
- **模拟引擎与 UI 的解耦**:
  - **决策**: 模拟计算引擎 `Simulator`、随机数生成器 `RNG`、因果记录 `CauseEffectLedger` 采用纯 TypeScript 实现，完全不依赖 Vue 组件或浏览器 DOM API。
  - **好处**: 使得核心逻辑可以直接在 Node.js 环境中通过命令行脚本执行。这为 `npm run validate:data` 和 `npm run sim:batch` 的开发打下了坚实的技术基础。
- **Hash 路由历史的选择**:
  - **决策**: 项目路由选用 `createWebHashHistory()`。
  - **好处**: 避免了在静态托管平台（如 GitHub Pages、S3 等）刷新子页面时出现 404 错误的问题，降低了运维门槛。
- **无 UI 框架设计**:
  - **决策**: 样式完全基于原生的 CSS 与 CSS 变量进行定制，没有加载 Element Plus 或 Ant Design 等重量级第三方组件库。
  - **好处**: 确保了极小的打包体积，并且使得布局在桌面端侧边栏与移动端底部菜单之间能极其平滑地实现响应式切换。

## 2. 数值平衡与参数校准 (RNG & Fiscal)
- **外部波动的 σ 值设定**:
  - **决策**: 我们将标准难度下的外部宏观波动标准差 $\sigma$ 锁定为 `0.012`。
  - **原因**: 运行批量测试后发现，如果设定过大，外部经济下行会造成级联崩盘，导致即使表现优异的市长也会因外部不可抗力触发财政破产结局；而设得过小则游戏挑战性过弱。
- **预测随机流隔离**:
  - **决策**: 四季度蒙特卡洛预测使用基于 `hash(seed, quarter, "forecast")` 生成的独立随机流。
  - **好处**: 杜绝了“利用预测页面刷正式随机数状态”的作弊行为，确保了存档重现的一致性。

## 3. 保存与恢复机制
- **IndexedDB 存档隔离**:
  - **决策**: 存档的读写通过 `saveService` 进行封装，使用 Zod Schema 在反序列化时执行强制校验。
  - **好处**: 防范了导入外部 JSON 存档时可能造成的格式崩溃或恶意 HTML/Script 注入安全风险。
